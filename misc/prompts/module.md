Consider the module system that Simulabra uses. I believe it's unnecessary, and want to try replacing it with ES Modules. The main obstacle is Class' use of the `name` var, which creates a method on it. Instead, it should define a property, as a special case. That probably means defining another class like BVar but for a property, and using that for name. The codebase will have to be updated with where it uses name. I think we should be consistent across classes, so that means every slot as well. `name()` -> `name` everywhere.
